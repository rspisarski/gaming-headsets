# Comprehensive Gaming Headset Features List

This is a complete reference for all filterable features that a gaming headset can have. Use this to build your data schema and filtering system.

---

## 1. CONNECTIVITY & WIRELESS

### Connection Types (can have multiple)
- **Wired 3.5mm** - Standard analog audio jack
- **USB-A** - USB Type-A connection (common on PC)
- **USB-C** - USB Type-C connection (modern devices)
- **2.4GHz Wireless** - Proprietary wireless dongle (low latency)
- **Bluetooth** - Standard Bluetooth wireless
- **Simultaneous Connections** - Can connect to multiple devices at once (e.g., 2.4GHz + Bluetooth)

### Wireless Specifications (if applicable)
- **Battery Life** - Hours of use on single charge (e.g., 15, 30, 38, 80 hours)
- **Quick Charge** - Minutes for X hours of use
- **Charging Type** - USB-C, USB-A, Micro-USB, Wireless charging
- **Swappable Batteries** - Hot-swappable battery system
- **Wireless Range** - Distance from source (typically 10-40 feet)
- **Latency** - Wireless delay in milliseconds (lower is better)

## 3. AUDIO SPECIFICATIONS

### Driver Information
- **Driver Size** - Diameter in mm (e.g., 40mm, 50mm, 53mm)
- **Driver Type** - Dynamic, Planar Magnetic, Electrostatic, Hybrid, Graphene
- **Multi-layer Drivers** - Dual-chamber or multi-layer driver design

### Sound Characteristics
- **Frequency Response** - Range in Hz (e.g., 20Hz - 20kHz, 10Hz - 40kHz)
- **Impedance** - Electrical resistance in ohms (e.g., 16立, 32立, 64立)
- **Sensitivity** - Volume efficiency in dB SPL/mW (e.g., 98 dB, 110 dB)
- **THD (Total Harmonic Distortion)** - Audio distortion percentage

### Surround Sound & Spatial Audio
- **Stereo** - Standard left/right audio
- **Virtual 5.1 Surround**
- **Virtual 7.1 Surround**
- **True 7.1 Surround** (multiple discrete drivers)
- **Dolby Atmos for Headphones**
- **DTS:X 2.0**
- **Windows Sonic**
- **THX Spatial Audio**
- **PlayStation Tempest 3D Audio** compatible
- **Proprietary Spatial Audio** (brand-specific like HyperX Spatial Audio)

### Headphone Type
- **Closed-back** - Sealed design, better noise isolation
- **Open-back** - Open design, wider soundstage, less isolation
- **Semi-open** - Hybrid design

---

## 4. MICROPHONE FEATURES

### Microphone Type
- **Boom Mic** - Attached boom arm microphone
- **Retractable Mic** - Mic retracts into earcup
- **Detachable Mic** - Can be completely removed
- **Inline Mic** - Mic on cable
- **Dual Microphone Array** - Multiple mics for better clarity
- **No Microphone** - Headphones only (not a headset)

### Microphone Specs
- **Frequency Response** - Mic frequency range (e.g., 100Hz - 10kHz)
- **Sensitivity** - Mic pickup sensitivity (dB)
- **Pickup Pattern** - Unidirectional, Bidirectional, Omnidirectional, Cardioid

### Microphone Features
- **Noise Cancellation** - Filters background noise
- **AI Noise Cancellation** - Advanced AI-powered noise filtering
- **Pop Filter** - Built-in pop filter
- **Sidetone** - Hear your own voice in the headphones
- **Mute Button** - Physical mute toggle
- **Auto-mute** - Mutes when mic is flipped up/retracted
- **LED Mute Indicator** - Light shows when muted
- **Monitoring** - Real-time mic monitoring
- **Broadcast Quality** - Studio-grade microphone quality

- **Plastic Frame**
- **Steel Frame**

- **PVD-coated Steel** - Premium corrosion-resistant coating
- **Carbon Fiber** - Lightweight and strong

## DATA SCHEMA RECOMMENDATIONS

### Boolean Fields (true/false)
- wireless
- wired
- bluetooth
- noise_cancellation (ANC)
- rgb_lighting
- detachable_mic
- replaceable_earpads
- software_support
- simultaneous_connections
- sidetone
- glasses_friendly
- foldable

### Numeric Fields
- price (decimal)
- battery_life (integer, hours)
- weight (integer, grams)
- driver_size (integer, mm)
- cable_length (decimal, feet/meters)
- warranty_years (integer)

### String/Enum Fields
- brand (string)
- microphone_type (enum: boom, retractable, detachable, inline, none)
- surround_sound (enum: stereo, virtual_5.1, virtual_7.1, dolby_atmos, dts_x, spatial_audio)
- ear_cup_type (enum: over-ear, on-ear)
- headphone_type (enum: closed-back, open-back, semi-open)
- frame_material (enum: plastic, steel, aluminum, carbon_fiber)
- cushion_material (enum: memory_foam, velour, leatherette, leather, microfiber)

### Array Fields (multiple values)
- connectivity (array: ["3.5mm", "USB-C", "2.4GHz", "Bluetooth"])
- compatibility (array: ["PC", "PS5", "Xbox Series X|S", "Switch", "Mobile"])
- included_accessories (array: ["Carrying case", "USB cable", "Extra ear pads"])
- certifications (array: ["Hi-Res Audio", "Discord Certified"])

### Text Fields
- frequency_response (string, e.g., "20Hz - 20kHz")
- impedance (string, e.g., "32立")
- description (text, SEO description)
- key_features (array of strings or text)

## NOTES FOR IMPLEMENTATION

### Data Quality
- Not all headsets will have all features - that's OK
- Focus on the most important/common features first
- Mark fields as "Not specified" or null when data is unavailable
- Verify specs against manufacturer websites

### User Experience
- Too many filters overwhelms users - prioritize the top 8-10
- Use progressive disclosure (show "Advanced filters" button)
- Show count of results as filters are applied
- Allow "Clear all filters" option

### Search Strategy
- Users should be able to search by model name, brand
- Consider fuzzy search for typos
- "Quick filters" for common use cases ("Best for PS5", "Under $100", "Wireless only")
